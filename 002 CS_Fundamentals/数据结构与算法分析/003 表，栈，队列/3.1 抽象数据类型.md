**程序设计的基本法则**之一是例程不应超过一页

一页代码 = 人眼**一次能看完**
→ 不需要翻页、不需要上下滚动
→ 大脑**一次就能理解**这个函数在干什么

行数过多会难以理解，难以修改

行为上强迫大功能拆成小功能

**抽象数据类型**（abstract data type ADT）
是一些操作的集合
ADT = 只定义 “能做什么”，不定义 “怎么实现”

**抽象数据类型（ADT）**不是代码，不是结构，不是变量。
它 = 一组对外提供的操作 + 操作规则。

ADT = 接口 = 功能列表 = 操作集合

比如 栈的 ADT 就是 **4 个操作**：

1. `Push(x)`：把 x 压入栈
2. `Pop()`：弹出栈顶元素
3. `Top()`：看栈顶是什么
4. `IsEmpty()`：栈空吗？
**这 4 个操作，就是 ADT。**

## 抽象数据类型与数据类型的关系

**数据类型**是一个值的集合和定义在该集合上一组操作的总称
分为 原子类型，结构类型，抽象数据类型
- 原子 其值不可分的数据类型
- 结构 其值可进一步拆分为若干成分的数据类型
- ADT 就是 “高级版、抽象版的数据类型”，所以它也叫数据类型。

数据类型 = 数据 + 可对它执行的操作集合

因为 **ADT 完全满足数据类型的定义**：

ADT = 数据 + 操作集合

只是：
- 普通类型：数据简单（int、float）
- ADT：数据复杂（表、栈、队列、树、图）

ADT 是一种数据类型，只不过是抽象的、用户定义的数据类型。
