## Compose注解

Compose 函数是构建 UI 的核心。它们使用 @Composable 注解来标记，代表该函数是一个可组合的 UI 单元。
就是用这个注解的可以互相调用

```
@Composable  
fun MyApp(  
    modifier: Modifier = Modifier,  
    names: List<String> = listOf("World", "Compose")  
) {  
    Column(modifier = modifier.padding(vertical = 4.dp)) {  
        for (name in names) {  
            Greeting(name = name)  
        }  
    }  
}
```

## Preview注解

不用运行模拟器就可以观察Compose函数的效果，但缺点是不接受带参数的可组合函数

```
@Preview(showBackground = true)  
@Composable  
fun GreetingPreview() {  
    BasicsCodelabTheme {  
        MyApp()  
    }  
}
```


## 基础代码解析

```
class MainActivity : ComponentActivity() {  
    override fun onCreate(savedInstanceState: Bundle?) {  
        super.onCreate(savedInstanceState)  
        enableEdgeToEdge()  
        setContent {  
            BasicsCodelabTheme {  
                Scaffold(modifier = Modifier.fillMaxSize()) { innerPadding ->  
                    MyApp(  
                        modifier = Modifier.padding(innerPadding)  
                    )  
                }  
            }        }    }  
}
```

### onCreate()函数

构造函数的加强版，系统帮我调用，用于初始化界面，加载ui

系统会先创建类（new），然后再调用onCreate

我不能自己new Activity

super .onCreate(savedInstanceState)调用父类的方法

#### savedInstanceState: Bundle?

`Bundle`：安卓的**键值对容器**，类似 `HashMap`，存字符串、数字等简单数据
第一次创建时为空(null)

页面被系统销毁重建时（比如**旋转屏幕、内存不够杀后台**），里面存着你之前的数据。

- 页面快被销毁 → 系统调用 `onSaveInstanceState(outState: Bundle)` 存数据
- 页面重建 → 调用 `onCreate(savedInstanceState)`，把刚才存的给你


### enableEdgeToEdge()

**让你的 APP 界面「全屏延伸」，铺满手机屏幕，顶到状态栏和导航栏边缘**，不再有黑边 / 白边。

- **不开启**：你的画挂在墙上，和天花板**留一条缝**
- **开启**：你的画**直接贴到天花板**，全屏铺满

以前的安卓 App：

- 顶部**状态栏**（时间、电量那一条）是黑的 / 白的
- 界面和状态栏之间**有间隔**

加了 `enableEdgeToEdge()` 后：

- **你的界面直接延伸到屏幕最顶端、最底端**
- 状态栏变成**透明 / 半透明**，覆盖在你的界面上
- 视觉更美观、更现代（现在所有新 App 都这么做）


### setContent

setContent = 告诉 Activity：我要用 Compose 来渲染界面，这里就是页面内容

#### 本质是什么？

- 它是 `ComponentActivity` 里的方法
- 大括号 `{}` 里面，就是**Compose 作用域**
- 只有在这个大括号里，才能写 `@Composable` 函数（Text、Button、Column 等）

真实执行顺序
1. 调用 `setContent()`
2. setContent **内部**自动执行你给的 `{ ... }`
3. 渲染 Compose UI

所以：

**不是 setContent 跑完再执行 {}，是 setContent 内部主动调用 {}**


### BasicsCodelabTheme

**它就是你整个 App 的 “皮肤 / 主题包”**
控制：颜色、字体、文字大小、黑暗模式、整体风格，**不是必须，但官方推荐用**。

包裹所有UI

它本质就是一个 **Compose 函数**，自动给界面设置：

- 颜色（主色、背景色、文字色）
- 字体样式
- 明暗模式（白天 / 黑夜）
- 形状（圆角、按钮样式）


### Scaffold

直译：**脚手架**

通俗讲：**给页面搭一个标准外壳**

就像网页里的 `<body>`，帮你处理好：

- 顶部状态栏（时间、电量）
- 底部返回条
    
    不用你自己算边距、防止内容被挡住。

`Scaffold` 是 Compose 官方提供的**页面基础布局壳子**，专门帮你处理手机自带的系统 UI：

顶部状态栏（时间、电量、信号）、底部导航条（返回、主页）。

它本质是一个 `@Composable` 函数，和 `Column`/`Row` 一样，只是功能更强。

#### 本质

一个可组合函数，用来搭建**标准安卓页面结构**。

传统 XML 里没有对应控件，你要自己手动写布局、算边距；

Compose 里直接用它，**自动适配状态栏、导航栏**。

#### 作用
1. 自动预留顶部安全区域，内容不会被状态栏盖住
2. 自动预留底部安全区域，内容不会被底部导航条盖住
3. 可以自带顶部标题栏、底部导航栏（你这里没用到）


#### modifier = Modifier.fillMaxSize()

`Modifier` = **修饰符**，专门用来控制组件的：大小、位置、边距、背景、点击等。

所有 Compose 控件都靠它改样式


#### `fillMaxSize()`

- 直译：填满最大可用空间
- 作用：让当前这个 `Scaffold` **占满整个手机屏幕**，上下左右铺满

为什么要加

不加的话，Scaffold 只会包裹内部内容大小，不会全屏；

加上后，页面背景、内容区域才是全屏。


### { innerPadding -> ... }

（lambda 代码块，重点）
和 `setContent { }` 完全一样：

- 大括号是**传给 Scaffold 的一段 UI 回调代码**
- Scaffold 内部会执行这段代码，用来渲染页面主体内容

- 是 Scaffold **自动计算出来的安全内边距**
- 里面包含：顶部要空多少 dp、底部要空多少 dp
- 作用：避开状态栏和底部导航栏，防止内容被遮挡



### MyApp(  modifier = Modifier.padding(innerPadding)

**自己定义的 @Composable 函数**，是真正显示业务内容的 UI。



## Compose 入门

Android 使用@Composable注解来表示可组合函数，从而定义UI

@Preview 注解标记所有无参数可组合函数或采用默认形参的函数，然后构建项目即可


### 微调界面

```
@Composable
fun Greeting(name: String, modifier: Modifier = Modifier) {
    Surface(color = MaterialTheme.colorScheme.primary) {
        Text(
            text = "Hello $name!",
            modifier = modifier
        )
    }
}

```

@Composable
- 作用：**告诉 Compose 编译器，这个函数是用来画界面的**
- 只有加了 `@Composable`，函数里面才能写 `Surface`、`Text`、`Column` 等 UI 组件
- 你可以把它理解成：**我是一个 UI 组件**

 fun Greeting(...)
- `fun` = 定义函数
- `Greeting` = 函数名（自定义的，随便取）
- 作用：**这是一个可复用的 UI 组件**，你可以在任何页面调用它

 modifier: Modifier = Modifier

- 第二个参数，**修饰符**
- `Modifier` 用来控制：大小、边距、位置、背景、点击等
- `= Modifier` 是**默认值**
    
    - 不传参数也能运行
    - 外面可以传自己的修饰符进来
    
- 作用：**让这个组件更灵活，外面可以控制它的样式**



### Surface(...)

- **一个容器组件 = 一个盒子**
- 相当于一个白色矩形底板
- 可以设置：背景色、形状、边框、点击效果
- 作用：**包裹里面的内容，给它加一个背景**

Surface ≈ HTML 里的 `<div>`

#### 相同点

1. 都是**容器盒子**，用来包裹里面的内容（Text 对应 `<p>/<span>`）
2. 都能设置**背景颜色、大小、边距、圆角**
3. 都是块级布局，占一块区域

#### 细微区别（重点，作业理解用）

1. **Surface 是安卓 Material 设计专用盒子**
    
    自带**阴影、点击波纹、主题颜色**，专门给 UI 卡片、按钮背景用。
1. div 就是普通盒子，功能基础；Surface 是**增强版 div**


- Surface → `<div>`（容器、背景）
- Text → `<p>` / `<span>`（文字）
- Modifier → CSS（控制大小、边距、颜色）

**@Composable 函数 = HTML 标签**

- `Surface` → `<div>`
- `Text` → `<p>/<span>`
- `Button` → `<button>`
- `Column/Row` → flex 布局


### 修饰符 modifier

对应html中的 style=""
干的是css（样式表）的事情

```
Text( text = "Hello $name!", modifier = modifier.padding(24.dp) )
```

```
<p style="padding: 24px;">Hello name!</p>
```


### 重复使用可组合项

就像使用html, 不会全些在div中而是分为html head body之类的

三条规则
- **每个自定义组件都要加 `modifier: Modifier = Modifier`**
- **默认值 = Modifier**（不传也能用）
- **把这个 modifier 传给组件内部最外层的盒子**

### Compose中的状态

使用 onClick来定义触发函数
然后可以借此来转递状态


```
@Composable
fun Greeting(name: String, modifier: Modifier = Modifier) {
    val expanded = remember { mutableStateOf(false) }
    val extraPadding = if (expanded.value) 48.dp else 0.dp
    Surface(
        color = MaterialTheme.colorScheme.primary,
        modifier = modifier.padding(vertical = 4.dp, horizontal = 8.dp)
    ) {
        Row(modifier = Modifier.padding(24.dp)) {
            Column(
                modifier = Modifier
                    .weight(1f)
                    .padding(bottom = extraPadding)
            ) {
                Text(text = "Hello ")
                Text(text = name)
            }
            ElevatedButton(
                onClick = { expanded.value = !expanded.value }
            ) {
                Text(if (expanded.value) "Show less" else "Show more")
            }
        }
    }
}
```
